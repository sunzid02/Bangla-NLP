# Per-Paper Verification Checklist

**Mandatory.** Run this checklist for every paper **before** adding it to a batch or to
any artifact. (Field definitions and publication-type rules live in `AGENTS.md`.)

## Checklist

Mark each item ✅ verified, or `Not clearly stated` if it cannot be confirmed:

- [ ] Title verified
- [ ] Authors verified
- [ ] Year verified
- [ ] Official venue verified
- [ ] Publication type verified
- [ ] ACL Anthology URL verified
- [ ] PDF checked
- [ ] Dataset verified
- [ ] Dataset size verified (if stated)
- [ ] Task type verified
- [ ] Method / model verified
- [ ] Evaluation metric verified
- [ ] Main result verified
- [ ] Contribution verified
- [ ] Limitation verified
- [ ] Future work verified (if stated)
- [ ] Beginner Bangla summary created
- [ ] No duplicate paper found
- [ ] Bangla/Bengali is central to the paper

## Rules

- If any item cannot be verified, write **`Not clearly stated`**.
- **Never infer missing fields.**
- If ACL Anthology and the PDF disagree, **report both** and mark **`Discrepancy found`**.
